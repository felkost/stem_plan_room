# domain/ — правила шару

Чисте ядро: типи й дані плану. **Заборонені** імпорти three.js, React,
браузерних API — лише TypeScript-типи та константи.

## Система координат (Sweet Home 3D)

- Одиниці — **сантиметри**; світ three.js — метри (`cm()` в infrastructure ділить на 100).
- `x` — на схід, `y` — на південь (у three.js це вісь z), `elevation` — низ
  об'єкта над підлогою.
- `angle` — радіани **за годинниковою стрілкою**; 0 = фронт на південь (+y).
  Орієнтири: крісло обличчям на захід ≈ `π/2`, на схід ≈ `3π/2`.

## Еталон плану (layoutBaseline)

- Координати з sh3d-плану («каб інф.sh3d», перенесено 1:1) — **незмінні**.
  Snapshot-тест навмисно падає при будь-якій зміні; оновлення `npx vitest run -u`
  допустиме лише за свідомого рішення про зміну плану.
- Нові об'єкти (поза планом, як `POD_DESKS`) додавати можна: новий const →
  списки в `layoutValidation.test.ts` (межі кімнати, SAT-колізії, кількість) →
  окремий it-блок у `layoutBaseline.test.ts` (снапшот запишеться сам при
  першому запуску, без `-u`).

## Валідація

- SAT-перевірка колізій має shrink-допуски: столи 4 см (сусідні в ряду стоять
  упритул), крісла 1 см. Крісло «під столом» — це перетин габаритів,
  тримай зазор ≥ 0.5 см між footprint'ами.
