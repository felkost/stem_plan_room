# STEM-кабінет інформатики — 3D-візуалізація

## Призначення

Реалістична інтерактивна 3D-модель кабінету інформатики у браузері, побудована за планом
Sweet Home 3D (`каб інф.sh3d`). Проєкт створено для презентації STEM-кабінету
(методична комісія, відкриті уроки, сайт школи): відвідувач може «походити» сценою,
оглянути її з будь-якого боку та зберегти скріншот чи відеозапис.

Акцент — робототехніка: на тренувальному столі з бортиками анімований робот
**LEGO Mindstorms NXT** їде по чорній лінії (line following), поруч запарковані інші
роботи, а у шафі-вітрині — виставка роботів **NXT / SPIKE Prime** з коробками
конструкторів. Оформлення тренувального поля відтворює реальний стіл кабінету.

## Стек технологій

| Технологія | Роль у проєкті |
| --- | --- |
| [three.js](https://threejs.org) | 3D-рендеринг (WebGL): кімната, меблі, роботи, освітлення, тіні |
| [React 18](https://react.dev) | Інтерфейс користувача (панель керування, довідка, медіа-кнопки) |
| [TypeScript](https://www.typescriptlang.org) | Строга типізація всіх шарів |
| [Vite](https://vite.dev) | Сервер розробки та збірка продакшн-бандла |
| MediaRecorder API | Відеозапис сцени у WebM без сервера |
| [serve](https://www.npmjs.com/package/serve) | Роздача зібраної версії (локально та на Railway) |

## Можливості

- 🏫 Точна розстановка за планом: Г-подібна кімната з нішею вчителя, 13 комп'ютерних
  столів з моніторами та кріслами, зелена дошка й інтерактивна панель на фронтальній
  (північній) стіні, мережевий комутатор, зарядна станція Bluetti.

  стільниці зі світлого дуба на чорних сталевих О-рамах, світло-сірі крісла,
  вертикальні жалюзі на вікнах, пастельні плакати; у центрі — 4 групові столи
  зі стільцями (додані поза sh3d-планом, базове розташування не змінено).
- 🤖 Тренувальний стіл із бортиками (за оформленням реального): робот NXT із
  датчиком лінії їде по чорній замкненій лінії, колеса обертаються; ще два роботи
  запарковані на полі.
- 🗄️ Шафа-вітрина: роботи NXT і SPIKE, кубок, коробки конструкторів.
- ☀️ Реалістичне освітлення: сонце крізь вікна з м'якими тінями, стельові LED-панелі.
- 🎥 Пресети камери, автообертання, перемикач якості для слабших комп'ютерів.
- 📷 Скріншоти та відеозапис прямо з браузера, без жодного сервера.
- 📱 Мобільна версія (iPhone 13+, Android): сенсорне керування (палець — обертання,
  щипок — масштаб), тулбар-стрічка з прокручуванням, безпечні зони екрана з вирізом,

  «Поділитися», відео у MP4 для Safari.

## Технології та архітектура

Vite + React 18 + TypeScript + three.js. Код організовано за **Onion Architecture**:

```text
src/
  domain/           ← ядро: типи й дані плану (жодних залежностей)
  application/      ← сценарії використання та порти (інтерфейси)
  infrastructure/   ← реалізації портів: three.js-сцена, MediaRecorder, збереження
                      файлів (Web Share / download), профіль пристрою (device/)
  presentation/     ← React-компоненти (UI)
  main.tsx          ← composition root: збирає шари докупи
```

Правило залежностей: усі стрілки вказують до центру. `three` імпортується **лише**
в `infrastructure/`, React — лише в `presentation/`.

## Порядок запуску локально

Потрібен [Node.js](https://nodejs.org) версії 18 або новішої (`node -v` — перевірити).

1. Встановити залежності (один раз):

   ```bash
   npm install
   ```

2. Запустити сервер розробки:

   ```bash
   npm run dev
   ```

3. Відкрити у браузері <http://localhost:5173>.

Продакшн-збірка та її локальна перевірка:

```bash
npm run build      # перевірка типів + збірка у dist/
npm start          # роздати зібрану версію → http://localhost:3000
```

## Тести та перевірки під час push

Дані плану захищені тестами ([vitest](https://vitest.dev)):

- **Валідація розстановки** (`src/domain/__tests__/layoutValidation.test.ts`):
  меблі в межах Г-подібної кімнати, без суттєвих перетинів, монітори стоять на
  столах, вікна/двері — у межах своїх стін.
- **Еталон базового розташування** (`layoutBaseline.test.ts`): snapshot усіх
  координат плану. Будь-яка зміна `classroomLayout.ts` «завалює» тест — зміни
  в плануванні мають бути свідомими. Після узгодженої зміни еталон оновлюється:
  `npx vitest run -u`.

Запуск вручну: `npm test` (або `npm run verify` — типи + тести разом).

Перевірки виконуються автоматично:

1. **Перед push локально** — git-хук `.githooks/pre-push` (підключається сам
   через npm-скрипт `prepare` під час `npm install`). Якщо типи або тести
   не проходять — push скасовується.
2. **Після push на GitHub** — workflow `.github/workflows/ci.yml`:
   типи → тести → продакшн-збірка.

## Розгортання на Railway (результат в інтернеті)

1. Зареєструйтеся на [railway.app](https://railway.app) (можна через GitHub).
2. **New Project → Deploy from GitHub repo** → оберіть цей репозиторій.
3. Railway прочитає `railway.json` і сам виконає `npm run build` та запустить сервер —
   нічого налаштовувати не треба.
4. Відкрийте сервіс → **Settings → Networking → Generate Domain** — отримаєте публічне
   посилання виду `https://….up.railway.app`, яким можна ділитися з учнями та колегами.

Після кожного `git push` у гілку `main` Railway автоматично переrозгортає сайт.

## Керування

| Дія | Як |
| --- | --- |
| Обертати камеру | ліва кнопка миші + рух (один палець на сенсорному екрані) |
| Наблизити / віддалити | коліщатко миші — зум іде до курсора (щипок двома пальцями) |
| Зсунути сцену | права кнопка миші + рух (або середня кнопка) |
| Зупинити автообертання | просто почніть рухати сцену мишею — воно вимкнеться само |
| Готові ракурси | кнопки внизу: Загальний вид, Арена, Дошка, Стіл учителя, Вітрина |
| Скріншот | кнопка «📷 Скріншот» → PNG у папці «Завантаження» |
| Відеозапис | «⏺ Запис відео» → «⏹ Зупинити» → WebM у папці «Завантаження» |

Порада: якщо на шкільному комп'ютері сцена «гальмує», натисніть кнопку якості —
вона перемкне тіні та роздільність у швидкий режим.
