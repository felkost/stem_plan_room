# CLAUDE.md — STEM-кабінет інформатики (3D)

Інтерактивна 3D-візуалізація кабінету інформатики за планом Sweet Home 3D
(«каб інф.sh3d»). Vite + React 18 + TypeScript + three.js. Деплой — Railway
(`railway.json`, роздача `dist/` через `serve`).

## Handoff між сесіями (обов'язково)

- **На старті сесії прочитай [memory.md](memory.md)** — там поточний стан,
  виконані етапи, карта кодової бази та перевірені прийоми. Не аналізуй
  кодову базу заново.
- **Після завершення суттєвого етапу онови memory.md**: перезапиши секцію
  «Поточний стан», додай рядок у «Журнал етапів» (новіші — зверху).
  Дрібні правки (тайпо, коментарі) журналювати не треба.

## Інженерні відкриття — скіл engineering-insights

Технічні відкриття (граблі, квіркі залежностей, неочевидні рішення) живуть в
`insights.md` кожного Onion-шару (`src/domain|application|infrastructure|presentation`)
і ведуться скілом `engineering-insights` (`.claude/skills/engineering-insights/`):

- **Початок роботи із шаром** → Mode C: прочитай `insights.md` шару, наведи
  2–5 релевантних записів.
- **Неочевидне відкриття посеред сесії** → Mode A: одразу задокументуй
  (append-only, лише Edit, формат — format.md скіла).
- **Кінець сесії > 30 хв із реальною проблемою** → Mode B: wrap-up чекліст.

Розподіл знань (не дублювати): CLAUDE.md — постійні правила; insights.md —
датовані відкриття по шарах; memory.md — стан, журнал етапів і крос-шарові
прийоми процесу.

## Команди

- `npm run dev` — дев-сервер (порт 5173); для тесту з телефона: `npm run dev -- --host`
- `npm run verify` — typecheck + vitest (його ж жене pre-push хук)
- `npx vitest run -u` — оновити еталонні знімки розташування
  (**лише за свідомої зміни плану** — див. src/domain/CLAUDE.md)
- `npm run build` — продакшн-збірка (`tsc --noEmit && vite build`)

## Архітектура — Onion, правило залежностей до центру

- `src/domain/` — чисті типи й дані плану; жодних імпортів three/React/браузерних API
- `src/application/` — сценарії використання та порти (інтерфейси)
- `src/infrastructure/` — реалізації портів: three.js-сцена, MediaRecorder,
  збереження файлів (Web Share / download), профіль пристрою (`device/`)
- `src/presentation/` — лише React-компоненти та styles.css
- `src/main.tsx` — composition root: детекція пристрою та впорскування залежностей
  відбуваються тільки тут

`three` імпортується **лише** в `infrastructure/`, React — **лише** в `presentation/`.
Нову логіку з браузерними API виноси в infrastructure за порт, а рішення-правила —
у чисті функції (їх покривай юніт-тестами без DOM).

## Ключові інваріанти

- Базове розташування об'єктів із sh3d-плану **незмінне** (захищено snapshot-тестом).
  Нові об'єкти додавати можна — див. чекліст у src/domain/CLAUDE.md.
- Усі текстури — процедурні canvas-текстури; жодних зовнішніх ассетів/CDN.
- Мова UI, коментарів і повідомлень комітів у коді — українська
  (коміт-меседжі — англійською, як у git-історії).

## Специфіка середовища

- Windows 10 + PowerShell 5.1. `gh` CLI **не встановлено** — GitHub через
  MCP-інструменти (create_pull_request тощо).
- pre-push хук виконує `npm run verify` — пуш із червоними тестами не пройде.
- Скріншот панелі браузера може падати по таймауту (панель прихована,
  `document.hidden` заморожує rAF/ResizeObserver) — обхід описано
  в memory.md → «Прийоми та граблі».
